Lab 01: Responsive Design

Excercise 1. Adaptable font size

In the stylesheet, change all the font sizes to use ems.

Try to keep all your font sizes the same as before, test in the browser to confirm that.

Notes:

Instead of using px for font sizes, use the unit of measurement em. An em is a unit equal to your default/base font size.

In most browsers, the default/base font size is 16px, in this case 1em = 16px. Everything is then measured based on this. This makes your website more scalable for all devices.

Note that once you define a main em size for fonts in a given container, it becomes the new base font size for the elements nested inside.Therefore you might want to leave CSS comments after each change to remind you what the original pixel sizes were.

Exercise 2. Page and element width

In the stylesheet of this page, change all the widths from px to percentages for the following boxes.

For the wrapper, set it to 100% width but limit its width to less than 960px, and greater than 350px.

For the content elements inside wrapper, give appropriate width percentage so that they fits nicely inside wrapper

Notes:

Fluid layouts should respond to the width of the browser window, which in most mobile devices is the width of the screen.

In order to achieve this, the page content widths should be defined as percentages

For a layout to be fluid, widths of all box elements must also be defined as percentages instead px.

Within a container, the total widths of all its children should add up to 100%.

Implementing Adaptive Design

The adaptive design approach means using the same markup but adapting the CSS to various viewports, using media queries. You can do adaptive layouts without using a fluid grid. Ours is already fluid, but the following techniques can be used with a fixed-width site.

The first thing you need to do is determine the browser widths at which your layout will need to change.

Exercise 5. Deciding Breakpoints

Now the layout is scalable. but when it scales on a small screen the content might get too small. When the screen size decreases, at some point we may wish to use a different layout instead of placing all elements horizontally.

Resize the page, try to find the points where you feel like some or all of the content is getting too tight horizontally. Determine how wide the browser window is using the dev tool at this point and that’s your first breakpoint.

Keep a record of the breakpoint and then think about what needs to go where for a layout beyond it.

Exercise 6. Adding Media Queries

Write a media query that tests for screen and your breakpoint width defined in exercise 5 above.

Write some rules inside your media query that change the layout and resize things to provide a better user experience at this size.

Notes:

A media query is something you wrap around a set CSS rules that tests for certain things going on in the browser, and applies those rules if the query is positive.

Media query example:

@media screen and (max-width: 360px) {
wrapper p {
font-size: 1.25em; /* =20px */
}
}

Exercise 6. Adding viewport

Add the viewport meta tag to your index page.

Notes:

Why viewport now??? Well, you never know, but some devices/browser may ignore your responsiveness and show your webApp stretched.

So it doesn't hurt if we add the viewport just in case.
